# Release Code Generation and Management

## Overview

Release codes are unique, single-use codes provided to customers who have completed all financing payments. These codes permanently remove all device restrictions, allowing customers to uninstall the Device Admin App and gain full control of their devices.

## Purpose

Release codes serve multiple purposes:
- **Verification:** Confirm that all payments have been completed
- **Security:** Prevent unauthorized device release
- **Audit Trail:** Track when and how devices were released
- **Customer Satisfaction:** Provide clear completion of financing agreement

## Release Code Requirements

### Format Specifications

**Code Structure:**
- **Length:** 12 characters
- **Character Set:** Alphanumeric (A-Z, 0-9)
- **Case:** Uppercase only
- **Format:** XXXX-XXXX-XXXX (with hyphens for readability)
- **Example:** `A7K9-M2P4-X8Q1`

**Security Requirements:**
- Cryptographically random generation
- No predictable patterns
- Unique per device
- Single-use only
- Time-limited validity (7 days)

### Code Properties

Each release code must have:
- **Device ID:** Associated device identifier
- **Customer ID:** Associated customer identifier
- **Generation Date:** When code was created
- **Expiry Date:** When code expires (7 days from generation)
- **Status:** ACTIVE, USED, EXPIRED, REVOKED
- **Generated By:** Staff member or system that created code
- **Used Date:** When code was successfully used (if applicable)

## Generation Process

### Prerequisites

Before generating a release code, verify:

1. **Payment Completion**
   - All scheduled payments are marked as PAID
   - Total paid amount equals total financing amount
   - No outstanding balance
   - No pending or failed transactions

2. **Device Status**
   - Device is registered in system
   - Device is active (not reported lost/stolen)
   - No outstanding issues or disputes
   - Customer account in good standing

3. **Authorization**
   - Request approved by authorized personnel
   - Customer identity verified
   - Proper documentation completed

**Step 1: Verify Payment Completion**

```sql
-- Check payment status
SELECT

    ps.paid_amount,
    ps.remaining_amount,

    COUNT(CASE WHEN i.status = 'OVERDUE' THEN 1 END) as overdue_payments
FROM devices d
JOIN payment_schedules ps ON d.device_id = ps.device_id

```

**Verification Criteria:**
- `remaining_amount` = 0
- `pending_payments` = 0
- `overdue_payments` = 0

**Step 2: Generate Release Code**

```python
import secrets
import string
from datetime import datetime, timedelta

def generate_release_code():
    """Generate a cryptographically secure release code"""
    # Character set: uppercase letters and digits, excluding similar-looking characters
    charset = string.ascii_uppercase + string.digits
    # Remove confusing characters: 0, O, 1, I, L
    charset = charset.replace('0', '').replace('O', '').replace('1', '').replace('I', '').replace('L', '')
    
    # Generate 12 random characters
    code = ''.join(secrets.choice(charset) for _ in range(12))
    
    # Format with hyphens for readability
    formatted_code = f"{code[0:4]}-{code[4:8]}-{code[8:12]}"
    
    return formatted_code

def create_release_code(device_id, customer_id, generated_by):
    """Create and store release code in database"""
    code = generate_release_code()
    
    release_code_data = {
        'code': code,
        'device_id': device_id,
        'customer_id': customer_id,
        'generated_date': datetime.utcnow(),
        'expiry_date': datetime.utcnow() + timedelta(days=7),
        'status': 'ACTIVE',
        'generated_by': generated_by,
        'used_date': None
    }
    
    # Store in database
    db.release_codes.insert(release_code_data)
    
    # Log the generation
    log_release_code_generation(device_id, customer_id, generated_by)
    
    return code
```

## Validation Process

### Device-Side Validation

When customer enters release code in app:

**Step 1: Format Validation**
```dart
bool validateCodeFormat(String code) {
  // Remove hyphens and spaces
  String cleanCode = code.replaceAll(RegExp(r'[-\s]'), '');
  
  // Check length
  if (cleanCode.length != 12) return false;
  
  // Check characters (alphanumeric only)
  if (!RegExp(r'^[A-Z0-9]+$').hasMatch(cleanCode)) return false;
  
  return true;
}
```

**Step 2: Backend Verification**

API Call: `POST /device/{deviceId}/verify-release-code`

**Step 3: Backend Validation Logic**

```javascript
async function verifyReleaseCode(deviceId, code) {
    // Step 1: Find code in database
    const releaseCode = await db.releaseCodes.findOne({
        code: code,
        deviceId: deviceId
    });
    
    if (!releaseCode) {
        return {
            valid: false,
            reason: 'INVALID_CODE',
            message: 'Release code not found'
        };
    }
    
    // Step 2: Check status
    if (releaseCode.status === 'USED') {
        return {
            valid: false,
            reason: 'ALREADY_USED',
            message: 'This code has already been used'
        };
    }
    
    if (releaseCode.status === 'REVOKED') {
        return {
            valid: false,
            reason: 'REVOKED',
            message: 'This code has been revoked'
        };
    }
    
    // Step 3: Check expiry
    if (new Date() > releaseCode.expiryDate) {
        // Auto-expire the code
        await db.releaseCodes.update(
            { code: code },
            { status: 'EXPIRED' }
        );
        
        return {
            valid: false,
            reason: 'EXPIRED',
            message: 'This code has expired. Please request a new one.'
        };
    }
    
    // Step 4: Verify payment completion (double-check)
    const paymentStatus = await checkPaymentStatus(deviceId);
    if (paymentStatus.remainingAmount > 0) {
        return {
            valid: false,
            reason: 'PAYMENT_INCOMPLETE',
            message: 'Payment not fully completed'
        };
    }
    
    // Step 5: Mark as used
    await db.releaseCodes.update(
        { code: code },
        { 
            status: 'USED',
            usedDate: new Date()
        }
    );
    
    // Step 6: Update device status
    await db.devices.update(
        { deviceId: deviceId },
        { 
            status: 'RELEASED',
            releaseDate: new Date()
        }
    );
    
    // Step 7: Log audit trail
    await logAudit('RELEASE_CODE_USED', {
        deviceId,
        code,
        timestamp: new Date()
    });
    
    return {
        valid: true,
        message: 'Release code verified successfully',
        deviceReleased: true,
        releaseDate: new Date()
    };
}
```

## Code Management

### Checking Code Status

**Query active codes:**
```sql
SELECT 
    rc.code,
    rc.device_id,
    rc.customer_id,
    rc.generated_date,
    rc.expiry_date,
    rc.status,
    rc.generated_by,
    d.device_model,
    c.customer_name
FROM release_codes rc
JOIN devices d ON rc.device_id = d.device_id
JOIN customers c ON rc.customer_id = c.customer_id
WHERE rc.status = 'ACTIVE'
ORDER BY rc.generated_date DESC;
```

**Query expired codes:**
```sql
SELECT 
    code,
    device_id,
    customer_id,
    generated_date,
    expiry_date
FROM release_codes
WHERE status = 'ACTIVE' 
  AND expiry_date < NOW();
```

### Revoking Codes

**When to revoke:**
- Payment dispute discovered
- Fraudulent activity detected
- Customer request (before use)
- Administrative error

**Revocation process:**
```sql
-- Revoke specific code
UPDATE release_codes
SET 
    status = 'REVOKED',
    revoked_date = NOW(),
    revoked_by = '<ADMIN_USER_ID>',
    revocation_reason = '<REASON>'
WHERE code = '<CODE>';

-- Update device status
UPDATE devices
SET 
    status = 'ACTIVE',
    release_eligible = FALSE
WHERE device_id = '<DEVICE_ID>';
```

### Regenerating Codes

**When to regenerate:**
- Original code expired
- Code lost by customer
- Code revoked and issue resolved

**Process:**
1. Verify original code status
2. Mark original as SUPERSEDED if active
3. Generate new code
4. Send to customer
5. Log regeneration in audit trail

```sql
-- Mark old code as superseded
UPDATE release_codes
SET status = 'SUPERSEDED'
WHERE device_id = '<DEVICE_ID>' 
  AND status = 'ACTIVE';

-- Generate new code (use generation process above)
```

## Security Considerations

### Code Security

**Storage:**
- Store codes hashed in audit logs
- Store plaintext only in release_codes table (encrypted at rest)
- Never log full codes in application logs
- Transmit only over encrypted channels (HTTPS, SMS)

**Generation:**
- Use cryptographically secure random number generator
- Avoid predictable patterns
- Ensure sufficient entropy (12 characters = ~60 bits)
- Exclude similar-looking characters (0/O, 1/I/L)

**Validation:**
- Rate limit validation attempts (max 5 per hour per device)
- Lock device after 10 failed attempts
- Alert backend on suspicious validation patterns
- Require online validation (no offline mode)

### Fraud Prevention

**Red Flags:**
- Multiple code generation requests for same device
- Code validation from different device than registered
- Validation attempts before code generation
- Rapid validation attempts (brute force)
- Code sharing between devices

**Monitoring:**
```sql
-- Detect suspicious activity
SELECT 
    device_id,
    COUNT(*) as validation_attempts,
    COUNT(DISTINCT code) as unique_codes_tried
FROM audit_logs
WHERE event = 'RELEASE_CODE_VALIDATION_ATTEMPT'
  AND timestamp > NOW() - INTERVAL '1 hour'
GROUP BY device_id
HAVING COUNT(*) > 5;
```

### FAQ for Customers

**Q: How long is my release code valid?**
A: 7 days from generation. Request a new one if expired.

**Q: Can I use the code multiple times?**
A: No, codes are single-use only.

**Q: What if I lose my code?**
A: Contact the store to request a new code.

A: No, codes are device-specific and should not be shared.

A: All restrictions are removed and you can uninstall the app.

A: No, but you can if you want. The device is fully yours.

A: Yes, factory reset will be enabled after release.

## Troubleshooting

### Code Not Working

**Customer reports code doesn't work:**

1. **Verify code format**
   - Check for typos
   - Ensure no extra spaces
   - Verify hyphens are correct

2. **Check code status in backend**
   ```sql
   SELECT * FROM release_codes WHERE code = '<CODE>';
   ```

3. **Verify device ID matches**
   - Code must be used on correct device
   - Check device ID in app settings

4. **Check expiry**
   - Generate new code if expired

5. **Check payment status**
   - Verify all payments completed
   - Check for pending transactions

### Code Not Received

**Customer didn't receive code:**

1. **Check delivery status**
   - Verify SMS was sent
   - Check email spam folder
   - Confirm phone number/email is correct

2. **Resend code**
   - Use alternative delivery method
   - Verify contact information first

3. **Generate new code if necessary**
   - If original lost or not delivered
   - Mark original as superseded

## Reporting

### Monthly Release Report

```sql
SELECT 
    DATE_TRUNC('month', generated_date) as month,
    COUNT(*) as codes_generated,
    COUNT(CASE WHEN status = 'USED' THEN 1 END) as codes_used,
    COUNT(CASE WHEN status = 'EXPIRED' THEN 1 END) as codes_expired,
    COUNT(CASE WHEN status = 'ACTIVE' THEN 1 END) as codes_active
FROM release_codes
GROUP BY DATE_TRUNC('month', generated_date)
ORDER BY month DESC;
```

### Device Release Statistics

```sql
SELECT 
    COUNT(*) as total_devices_released,
    AVG(EXTRACT(EPOCH FROM (release_date - purchase_date))/86400) as avg_days_to_payoff,
    MIN(release_date) as first_release,
    MAX(release_date) as latest_release
FROM devices
WHERE status = 'RELEASED';
```
